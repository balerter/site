**Balerter** - это **Alert Manager**. 

Основное предназначение Balerter - дать возможность как можно проще описать сложные правила алертинга.

Для простого Алерта, который срабатывает при превышении количества http запросов - достаточно будет даже встроенного функционала Grafana (оставим за скобками целесообразность этого подхода)
 
Однако, для более сложных случаев это не подойдет. Например, вы если вы хотите выбрать исторические данные из БД, затем сравнить их с текущими данными, сходить в сторонний сервис по API и затем уже принять решение об алертинге.

Именно для подобных кейсов и создан Balerter.

Вся логика описывается в lua-скриптах. 

Внутри скрипта вы можете:
- получить данные из различных источников, например Prometheus или Postgres (etc)
- произвести любые требуемые вам вычисления и принять решение об алерте 
- задать требуемый уровень алерту Success, Warning или Error
- настроить поведение, при котором уведомления будут отправляться в различные канали, в зависимости от уровня алерта. Например, warning уходит дежурному инжереру, а Error еще и руководителю
- создать и прикрепить график к уведомлению, чтобы можно быть сразу наглядно оценить ситуацию
