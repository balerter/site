В начале каждого скрипта можно указать мета-теги

```
-- @interval 10s
-- @ignore

local log = require('log')
...

```

Каждый мета-тег имеет вид `@<TAG_NAME> [<TAG_OPTIONS>]`

Все мета-теги должны быть строго в начале скрипта и располагаться в начале комментария.

Любая не пустая строка не-комментарий прервет обработку мета-тегов.

```
-- можно помещать комментарии без мета-тегов. они будут пропущены
-- @interval 10s (комментарий тут не допускается, перенесите его на другую строку, если он необходим)
--
-- пустая строка комментария тоже допустима


-- а так же допустимы пустые строки

local a = 10

-- @ignore
-- этот мета-тег не будет обработан, так как перед ним есть не пустая строка не-комментарий
```  

### Мета-теги

#### `@interval <TIME INTERVAL>`

Позволяет указать интервал, с которым будет запускаться скрипт.
Если не указан, используется значение по-умолчанию 60 секунд

Допустимые значения единиц времени: "ns", "us" (or "µs"), "ms", "s", "m", "h".

Вы можете использовать любые значения, однако стоит понимать, что запуск скрипта каждую микросекунду не кажется здравой идеей.

Пример:
```
-- @interval 10s
-- @interval 5m
```
                                    
#### `@ignore`

Если указан этот мета-тег, то скрипт не будет выполняться.

Например:
```
-- @ignore
```


#### `@name <SCRIPT NAME>`

Указать свое имя для скрипта. По умолчанию в качестве имени используется полный путь к файлу

Имя скрипта используется для выдачи отладочной информации и сообщений об ошибках в логах

Например:
```
-- @name script1
```

#### `@channels <CHANNEL NAME 1>,<CHANNEL NAME 2>,...`

Позволяет переопределить список каналов, куда будет производится отправка уведомлений этого скрипта

Названия каналов вы определяете в файле конфигурации

Например:
```
-- @channels slack-manager,email-ceo
```

#### Пример использования нескольких мета-тегов

```
-- multi meta example

-- @name MyScript
-- @interval 5m

-- @channels slack-ceo,telegram-ceo

...
```
